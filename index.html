⚡ How image compression works

Images (JPEG, PNG, RAW, HEIC, BMP, TIFF…) are compressed using WebP by default — WebP delivers 25–35% better quality per byte than JPEG at the same file size. After downloading, you'll get an option to convert back to your original format (JPEG, PNG, etc.) at the same compressed size.

PNG files stay as PNG using palette quantisation (pngquant-style). GIF and AVIF also stay native. The compressor runs a binary search across quality levels and resolution scales to find the best possible result within your target.

⚙️ Compression engines
Module Used for Why it's the best
sharp All images libvips-powered — fastest high-quality image processor; Lanczos3 resampling prevents aliasing
mozjpeg JPEG output Mozilla's JPEG encoder — trellis quantisation + optimised Huffman = 10–20% smaller than standard libjpeg
pngquant PNG Palette quantisation + Floyd–Steinberg dithering — 40–70% size reduction with minimal visual loss
WebP encoder JPEG-family images Google's WebP — 25–35% better quality/byte than JPEG; used as primary output for maximum quality at small sizes
pdf-lib PDF Re-saves with object streams + cross-reference compression; strips all metadata
archiver Documents Zip/deflate level 9 — best for DOCX/XLSX/PPTX (already zip-based) and plain text files
zlib Fallback Triple-pass: deflate-raw → gzip → deflate; picks whichever produces the smallest output
